# Commands — 智能家居 PM 工作流命令

命令将多个技能编排为端到端的工作流，通过单一的 `/` 触发执行完整的 PM 流程。

## 可用命令

| 命令 | 说明 | 时间 | 适用场景 |
|------|------|------|----------|
| `/write-prd` | 完整 PRD 开发流程（发现→方案→工程→输出→评审） | 2.5-3.5h | 新产品立项、重大功能改版 |
| `/discover` | 产品发现与验证流程（问题→假设→验证→决策） | 1-1.5h(轻量) / 2-5d(完整) | 探索新方向、验证需求假设 |

## 使用方式

```
/命令名 [参数]
示例：/write-prd 智能门锁 中国中高端家用
示例：/discover 带屏幕的智能音箱 "用户需要音箱上有屏幕"
```

## 设计原则

- **流程引导**：每个 Phase 明确输入/输出/使用的技能
- **决策点内置**：在关键节点要求做出 Go/No-Go 判断
- **时间盒控制**：每个 Phase 有建议时长，避免过度分析
- **证据驱动**：要求引用用户数据或验证结果，而非主观判断
