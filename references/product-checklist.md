# 智能家居产品健康度检查清单

## 一、PRD 完成度自检

### 1.1 产品定义

- [ ] 产品背景与目标清晰，有量化指标
- [ ] 目标用户画像完整（至少 2 个用户画像）
- [ ] 核心价值主张能用一句话表达（<20字）
- [ ] 产品范围边界明确（含本期范围和非目标说明）
- [ ] 成功指标（KPI）已定义，含测量方式

### 1.2 功能需求

- [ ] 核心功能有明确的用户故事（格式：作为/我希望/以便）
- [ ] 每个功能有可验证的验收标准（Given/When/Then）
- [ ] 所有功能标注了优先级（MoSCoW）
- [ ] 增值功能与核心功能区分清晰
- [ ] 无超出范围的功能混入

### 1.3 非功能需求

- [ ] 性能要求明确（响应时间、功耗、续航等）
- [ ] 安全要求覆盖（通信加密、固件安全、数据隐私）
- [ ] 兼容性要求明确（硬件兼容、OS兼容、协议兼容）
- [ ] 可用性要求清晰（离线工作、异常提醒、应急方式）
- [ ] 运维与监控要求已定义（在线率、告警延迟、日志保留）

### 1.4 流程与数据

- [ ] 主流程和异常流程均已覆盖
- [ ] 边界情况/极端场景已说明
- [ ] 数据字段定义完整（字段名、类型、说明）
- [ ] 数据流转路径清晰（端到端）
- [ ] 存储方案选型合理

---

## 二、技术方案评审 Checklist

### 2.1 通信方案

- [ ] 通信协议选型有明确依据（功耗/距离/设备量/成本）
- [ ] 芯片选型考虑了备选方案（≥2款芯片对比）
- [ ] 网关需求已明确（是否需要/网关类型）
- [ ] 离线工作能力已考虑
- [ ] 配网流程设计已完成

### 2.2 云端架构

- [ ] 架构方案与产品规模匹配（自建/IoT平台/混合）
- [ ] MQTT/HTTP 等协议选择有说明
- [ ] 数据库选型有依据（时序/关系/缓存）
- [ ] 消息推送方案已确定（FCM/APNs/国内厂商Push）
- [ ] OTA 升级方案已设计（全量/差分、回滚机制）

### 2.3 安全设计

- [ ] 设备身份认证方案就绪（证书/Token）
- [ ] 通信加密方案确定（TLS版本）
- [ ] 固件安全启动和签名验证已规划
- [ ] 用户数据隐私保护措施到位
- [ ] 安全审计和漏洞响应流程已定义

### 2.4 App 端方案

- [ ] App 功能模块与 PRD 功能对齐
- [ ] 支持平台明确（iOS/Android 最低版本）
- [ ] 语音助手集成方案确定（如需要）
- [ ] 第三方登录/账号体系已规划

---

## 三、上线前核对清单

### 3.1 认证与合规

- [ ] 目标市场强制认证已完成（CCC/SRRC/FCC/CE等）
- [ ] 无线频谱合规（SRRC/FCC 等）
- [ ] 行业标准检测已通过（如 GA 374 电子防盗锁）
- [ ] 隐私政策与应用权限说明已准备
- [ ] 产品标识（认证标志、能效等级）已合规

### 3.2 质量与测试

- [ ] 硬件可靠性测试完成（高低温/跌落/老化）
- [ ] 通信稳定性测试完成（丢包率/延迟/并发）
- [ ] 电池续航测试完成（与标称值偏差 <10%）
- [ ] App 兼容性测试完成（主流机型覆盖）
- [ ] 安全渗透测试已完成
- [ ] 用户验收测试（UAT）已通过

### 3.3 运维就绪

- [ ] 设备激活/配网成功率 ≥ 95%
- [ ] OTA 升级流程已演练
- [ ] 异常监控与告警通道已配置
- [ ] 客服 FAQ 和培训材料已准备
- [ ] 退换货和售后流程已就绪

### 3.4 数据与指标

- [ ] 核心埋点已上线并验证（参照 PRD 事件定义）
- [ ] 业务看板已配置（日活、激活、留存）
- [ ] 稳定性看板已配置（在线率、推送到达率）
- [ ] 异常告警阈值已设置
- [ ] 数据统计口径已对齐（前后端/BI 一致）

---

## 四、版本迭代检查

### 4.1 版本规划

- [ ] MVP 范围明确且可交付
- [ ] 后续版本有清晰的功能拆分
- [ ] 每个版本有预估时间节点
- [ ] 版本间兼容性已考虑（数据迁移、协议升级）

### 4.2 迭代复盘

- [ ] 上个版本 KPI 达成情况已复盘
- [ ] 用户反馈已收集并整理优先级
- [ ] 技术债务已记录并排期
- [ ] 竞品动态已更新至分析文档

---

## 五、文档维护检查

- [ ] PRD 与实际实现一致（无僵尸文档）
- [ ] 厂商/芯片信息已核对时效性
- [ ] API 文档/SDK 文档已更新
- [ ] 变更记录（CHANGELOG）已更新
- [ ] 已知问题和限制已在文档中标注
