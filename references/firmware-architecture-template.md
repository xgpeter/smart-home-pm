# 智能家居固件架构模板

> 本模板定义智能家居设备固件的架构设计规范，覆盖 OS、任务划分、存储、OTA、低功耗等核心模块。

## 1. 运行环境

### 1.1 RTOS 选型

| 对比维度 | FreeRTOS | Zephyr | 裸机(Super Loop) |
|----------|----------|--------|-------------------|
| 适用场景 | 中等复杂度设备 | 复杂设备、Matter | 简单传感器 |
| ROM 占用 | ~10KB | ~50KB | <2KB |
| RAM 占用 | ~2KB | ~10KB | <1KB |
| BLE/WiFi 协议栈 | 需自行集成 | 原生支持 | 第三方 SDK |
| 推荐 | ✅ 大多数设备 | Matter 设备 | 极简传感器 |

### 1.2 内存分区

```
Flash 布局 (4MB 示例):
┌───────────────────── 0x000000
│ Bootloader           (64KB, 受写保护)
├───────────────────── 0x010000
│ App Firmware (A)     (1.5MB, 当前运行)
├───────────────────── 0x190000
│ App Firmware (B)     (1.5MB, OTA 升级区)
├───────────────────── 0x310000
│ NVS (Non-Volatile)   (512KB, 配网信息+用户数据)
├───────────────────── 0x390000
│ Factory Data         (64KB, 设备证书+校准数据，只读)
├───────────────────── 0x3A0000
│ OTA Info + Scratch   (384KB, 升级状态+临时空间)
└───────────────────── 0x400000
```

## 2. 任务架构

### 2.1 任务划分（FreeRTOS 示例）

| 任务名 | 优先级 | 栈大小 | 周期/触发 | 职责 |
|--------|--------|--------|-----------|------|
| sys_monitor | 最高(5) | 2KB | 1s周期 | 看门狗喂狗、电量检测、系统健康 |
| comm_mqtt | 高(4) | 6KB | 事件驱动 | MQTT 连接、保活、消息收发 |
| sensor_task | 中(3) | 4KB | 中断/周期 | 指纹采集、门磁检测、触摸扫描 |
| motor_ctrl | 中(3) | 2KB | 事件驱动 | 电机控制（开锁/上锁） |
| led_audio | 低(2) | 2KB | 事件驱动 | LED 指示、语音播报、蜂鸣器 |
| ota_task | 低(2) | 8KB | 事件驱动 | OTA 下载、校验、升级 |
| idle_log | 最低(1) | 2KB | 10s周期 | 日志汇总批量上报 |

### 2.2 任务间通信

| 机制 | 场景 | 示例 |
|------|------|------|
| 消息队列(Queue) | 传感器→主控 | 指纹识别结果推送到 motor_ctrl |
| 信号量(Semaphore) | 同步 | OTA 完成通知 comm_mqtt 上报 |
| 事件组(Event Group) | 多条件等待 | 等门磁+指纹同时就绪 |
| 互斥锁(Mutex) | 临界区 | NVS Flash 写入保护 |

## 3. 状态机设计

### 3.1 设备全局状态机

```
                    ┌──────────┐
        上电 ──────→│  INIT    │
                    │ (自检)   │
                    └────┬─────┘
                         │ 自检通过
                    ┌────▼─────┐
              ┌─────│ IDLE     │
              │     │ (待机)   │
              │     └──┬──┬───┘
              │        │  │
      用户交互│   传感器│  │远程指令
              │   触发 │  │
              │   ┌────▼──▼───┐
              │   │  ACTIVE   │
              │   │ (执行动作)│
              │   └────┬──────┘
              │        │ 执行完成
              │        └──→ IDLE
              │
              └── 深度休眠 ──→ DEEP_SLEEP ──→ (唤醒)→ INIT
```

### 3.2 配网状态机

```
IDLE → BLE_ADVERTISING → PROVISIONING → CONNECTING → CONNECTED → REGISTERED
                                  │              │
                                  └── 超时120s ──┴── 重试3次失败 → IDLE
```

### 3.3 开锁执行状态机

```
IDLE → FINGERPRINT_SCAN → MATCHING → MOTOR_UNLOCK → LOCKED
  │         │(超时)          │(失败3次)     │(门未关)
  │         ▼                ▼              ▼
  └── IDLE ←── ALERT ←── IDLE + NOTIFY ←── ALERT + LOG
```

## 4. 低功耗设计

### 4.1 功耗模式定义

| 模式 | 电流 | 唤醒源 | 进入条件 |
|------|------|--------|----------|
| Active | ~80mA | — | 用户操作/通信中 |
| Modem Sleep | ~20mA | WiFi Beacon | 空闲 >10s |
| Light Sleep | ~0.8mA | GPIO/定时器 | 空闲 >60s |
| Deep Sleep | ~50μA | 触摸/门磁/定时器 | 空闲 >5min |

### 4.2 功耗优化策略

- **触摸唤醒**：深度睡眠期间仅保持触摸检测电路供电（<10μA）
- **BLE 广播占空比**：默认 100ms 间隔，10 分钟后降至 1s
- **WiFi DTIM**：设置 DTIM=3，每 300ms 唤醒一次收信标
- **批量上报**：非紧急数据累积后一次性 WiFi 连接上报
- **传感器轮询**：门磁/霍尔使用中断唤醒，不做周期轮询

## 5. OTA 升级

### 5.1 升级流程

```
云端推送OTA通知 → 设备下载固件(HTTPS) → MD5/SHA256校验
    → 写入B分区 → 设置启动标记 → 重启
    → Bootloader 检查标记 → 启动新固件 → 上报结果
    ↓ (启动失败3次)
    Bootloader 自动回滚到 A 分区
```

### 5.2 差分升级（可选）

| 方案 | 差分包大小 | CPU 开销 | 适用场景 |
|------|-----------|----------|----------|
| 全量升级 | 1.5MB | 低 | 所有设备 |
| 差分升级(bsdiff) | ~200KB | 中(需解压) | WiFi 设备 |
| 差分升级(HDiffPatch) | ~150KB | 低(RAM 友好) | 内存受限设备 |

### 5.3 安全要求

- 固件包必须使用厂商私钥签名，设备端用公钥验签
- 下载使用 HTTPS，证书固定(Certificate Pinning)
- 版本号必须单调递增，禁止降级（生产环境）
- OTA 过程中保持设备可用（门锁 OTA 期间禁止自动开锁）

## 6. 日志与诊断

### 6.1 日志分级

| 级别 | 场景 | 本地存储 | 云端上报 |
|------|------|----------|----------|
| ERROR | 系统崩溃、通信失败、硬件异常 | ✅ 永久 | ✅ 即时 |
| WARN | 低电量、重连、降级 | ✅ 100条循环 | 批量 |
| INFO | 开锁、配网、OTA | ✅ 50条循环 | 批量 |
| DEBUG | 调试信息 | 仅开发版本 | ❌ |

### 6.2 远程诊断

- 云端下发 `get_device_log` 服务调用获取最近 N 条日志
- 支持通过诊断模式开启 DEBUG 日志（有时效性，24h 自动关闭）
- 关键异常自动上报 Crash Dump（寄存器状态 + 调用栈）

## 7. 安全启动

```
上电 → BootROM(不可篡改) → 验证 Bootloader 签名
    → Bootloader 验证 App 签名
    → 启动 App
    → App 运行时安全: 栈保护、MPU 区域隔离
```

### 7.1 防攻击措施

| 攻击方式 | 防护措施 |
|----------|----------|
| 固件篡改 | 签名验证 + Secure Boot |
| 调试接口 | 量产熔断 JTAG，仅保留 SWD(需密钥认证) |
| 侧信道攻击 | 关键操作时间随机化 |
| 故障注入 | 电压/时钟异常检测 |
| 固件回滚 | 版本号单调递增 + 防回滚计数 |

## 8. 工厂产测

### 8.1 产测模式

设备上电后检测特定 GPIO 组合（或通过产测 App 发送指令）进入产测模式。

### 8.2 产测项

| 测试项 | 方法 | 合格标准 |
|--------|------|----------|
| 上电自检 | 读取各传感器 ID | 所有外设 I2C/SPI 通信正常 |
| 射频测试 | 定频发射，频谱仪测量 | 功率 ±1dBm，频率偏移 ±20ppm |
| 指纹模组 | 标准假手指测试 | 采集图像无坏点 |
| 电机 | 正反转测试 | 电流 ≤ 额定值，无堵转 |
| LED/蜂鸣器 | GPIO 逐个点亮 | 全部正常 |
| 电池电压 | ADC 采样 | 偏差 ≤ 0.1V |
| MAC 写入 | 写入设备唯一 MAC + SN | 写入成功且可读回 |
