## 2. 产品概述

### 2.1 产品背景与目标

随着智能家居市场快速发展，智能门锁作为家庭安防的第一道防线，2026 年中国市场出货量预计突破 2500 万套。本产品定位中高端家用智能门锁，以"安全 + 便捷 + 智能联动"为核心，面向 25-40 岁品质家庭用户。

### 2.2 目标用户画像

**P1 — 年轻家庭（25-35岁）**
- 场景：双职工、有小孩、偶尔老人来访
- 痛点：忘记带钥匙、保姆/老人需要临时进门权限
- 诉求：指纹/人脸快速进门、临时密码、开锁记录

**P2 — 科技爱好者（28-40岁）**
- 场景：全屋智能、HomeKit/米家生态
- 痛点：门锁是唯一没接入智能家居的设备
- 诉求：回家场景联动、语音查询状态

### 2.3 核心价值主张

> "安全守护，一触即开" — 半导体指纹 + 3D 人脸 + 虚位密码，8 种开锁方式，TLS 1.3 全链路加密

### 2.4 范围边界

**本期范围：**
- 家用防盗门智能门锁（标准 6068 锁体）
- WiFi + BLE 双模通信
- App（iOS 14+ / Android 10+）配套管理
- 米家 & HomeKit 双生态接入

**非目标：**
- 酒店/B 端门禁、公寓 SaaS 平台
- Zigbee/Z-Wave 版本（V2 规划）
- 海外多地区认证（本期仅中国）

### 2.5 成功指标（KPI）

| 指标 | 目标值 | 统计口径 | 观察周期 |
|------|--------|----------|----------|
| 指纹识别成功率 | ≥ 98% | (成功次数/总尝试次数)×100%，排除未注册指纹 | 周 |
| 人脸识别成功率 | ≥ 95% | 正常光照条件下 | 周 |
| 平均开锁时间 | ≤ 1.5s | 从触摸唤醒到锁舌收回 | 日 |
| 首周激活成功率 | ≥ 92% | 蓝牙配网成功/设备激活总数 | 周 |
| 设备在线率 | ≥ 99.5% | 设备在线时长/总时长 | 月 |
| App Store 评分 | ≥ 4.5 | — | 月 |

## 3. 功能需求

### 3.1 核心功能（Must）

#### F-01 指纹开锁

**功能描述**：半导体电容式指纹识别，支持 50 枚指纹存储，活体检测防止假指纹。

**用户故事**：作为住户，我希望通过指纹快速开锁，以便不用携带钥匙便捷进出。

**验收标准**：
```
Given 用户已录入指纹且设备处于待机状态
When 用户触摸指纹识别区
Then 1.0s 内完成识别并驱动电机开锁
And 连续 5 次失败后禁用指纹 3 分钟，推送告警
And 假指纹（硅胶/明胶）被拒识
```

**优先级**：Must

---

#### F-02 3D 人脸识别开锁

**功能描述**：双目结构光 + 红外摄像头，支持暗光（0 lux）识别，活体检测防照片/视频攻击。

**用户故事**：作为双手提满物品的住户，我希望通过人脸识别无接触开锁，以便不用放下东西即可进门。

**验收标准**：
```
Given 用户已注册人脸且距离门锁 40-80cm
When 用户面向门锁
Then 1.5s 内完成识别并开锁
And 0 lux 全黑环境下可正常工作
And 照片/视频/3D 面具攻击被阻止
```

**优先级**：Must

---

#### F-03 密码开锁

**功能描述**：支持 6-12 位虚位密码，最长 32 位（含虚位），连续 5 次错误锁定 5 分钟。

**用户故事**：作为住户，我希望设置安全密码开锁，以便在指纹磨损或人脸遮挡时作为备用。

**验收标准**：
```
Given 用户已设置 6-12 位密码
When 输入正确密码（可含前后任意虚位）
Then 密码验证通过并开锁
And 连续 5 次错误本地锁定 5 分钟，推送告警
```

**优先级**：Must

---

#### F-04 异常告警推送

**功能描述**：撬锁、多次验证失败、低电量、防拆检测触发实时告警推送。

**用户故事**：作为住户，我希望及时收到门锁异常告警，以便第一时间处理安全问题。

**验收标准**：
```
Given 门锁检测到异常（撬锁/5次错误/低电量/防拆）
When 异常发生
Then 5 秒内本地警报触发，App 推送到达率 ≥ 95%
And 云端记录完整告警事件
```

**优先级**：Must

---

### 3.2 增值功能（Should）

#### F-05 临时密码

支持通过 App 生成一次性/限时/限次临时密码，适用于访客、保洁、快递场景。密码到期自动失效。

**优先级**：Should

#### F-06 猫眼可视对讲

集成 1080P 广角摄像头（160°），支持按铃唤醒 App 双向通话、PIR 逗留抓拍（3 米范围）。

**优先级**：Should

#### F-07 开锁记录查询

记录所有开锁事件（方式、用户、时间、抓拍照片），支持 90 天历史查询和按维度筛选。

**优先级**：Should

### 3.3 增值功能（Could）

#### F-08 智能场景联动

开锁触发回家场景（开灯/空调/窗帘），离家场景（一键布防），支持米家和 HomeKit 快捷指令。

**优先级**：Could (V1.2)

## 4. 非功能需求

### 4.1 性能要求

| 指标 | 目标值 | 测试条件 |
|------|--------|----------|
| 指纹识别响应 | ≤ 1.0s | 从触摸到锁舌收回 |
| 人脸识别响应 | ≤ 1.5s | 正常光照 40-80cm |
| 密码验证响应 | ≤ 0.5s | — |
| 远程开锁延迟 | ≤ 3s | WiFi 正常网络环境 |
| 电池续航 | ≥ 12 个月 | 每天开锁 10 次 + 猫眼抓拍 20 次 |
| 待机功耗 | ≤ 80μA | 深度睡眠模式 |
| 电机扭矩 | ≥ 2.5 N·m | 标准 6068 锁体 |

### 4.2 安全要求

| 层面 | 要求 |
|------|------|
| 生物识别 | 指纹活体检测（拒假率 ≤ 0.01%），人脸 3D 结构光防照片/视频 |
| 通信加密 | TLS 1.3 全链路，MQTT over TLS，证书固定(Cert Pinning) |
| 固件安全 | 安全启动 + 签名校验 + 防回滚（版本单调递增） |
| 本地存储 | 指纹/人脸特征数据仅存储在设备安全区域，不上传云端 |
| 密钥管理 | 设备一机一密(预烧录)，access_token 24h 过期 |
| 物理安全 | 防撬检测、防猫眼开锁、面板防拆卸 |

### 4.3 兼容性要求

- **门厚**：40-120mm
- **锁体**：标准 6068 锁体，兼容霸王锁体（选配）
- **门类型**：木门、防盗门、铜门
- **App**：iOS 14+ / Android 10+

### 4.4 可用性要求

- 离线开锁：指纹/密码/机械钥匙不受 WiFi 断网影响
- 应急供电：Type-C 5V 应急供电接口（隐藏式）
- 低电量提醒：剩余 20% 语音提醒 + App 推送，剩余 5% 每次开门提醒
- 机械钥匙：保留物理钥匙孔（应急最后手段）

### 4.5 运维与监控要求

- 设备在线率 ≥ 99.5%
- OTA 升级成功率 ≥ 98%
- 云端 API 可用性 ≥ 99.9%
- 异常告警延迟 ≤ 5 秒
- 设备日志云端保留 ≥ 90 天
- 支持远程日志导出和诊断模式

## 5. 用户流程

### 5.1 首次安装配网

```
1. 安装师傅安装门锁 → 上电自检（LED 蓝灯闪烁=正常）
2. 用户下载 App → 注册登录
3. App 扫描门锁二维码 → BLE 连接 → 发送 WiFi SSID+密码
4. 门锁连 WiFi → MQTT 注册 → 云端激活
5. 设置管理员密码 → 录入指纹(≥2枚) → 录入人脸
6. 完成
```

### 5.2 日常开锁

```
触摸面板唤醒 → 验证指纹/人脸/密码 → 电机驱动开锁 → 语音反馈 → 门磁检测关门 → 自动上锁(可配置延时)
```

### 5.3 异常处理

| 异常 | 处理方式 |
|------|----------|
| 指纹失败 3 次 | 提示使用密码，推送通知 |
| 密码错误 5 次 | 锁定 5 分钟，推送告警 |
| 电量 ≤ 10% | 每次开锁语音提示，推送低电量 |
| 撬锁检测 | 本地警报 + 推送紧急通知 + 自动录像(如有猫眼) |
| WiFi 断开 | BLE 告警通知附近手机，离线不影响本地开锁 |

### 5.4 边界情况

| 边界场景 | 处理方式 |
|----------|----------|
| 门未关严 | 门磁检测 → 语音提示"门未关好" → 不上锁 → 30s 后再次提示 |
| 内侧反锁 | 反锁优先级最高，远程开锁/密码/指纹均无效，App 提示"已反锁" |
| 多人同时操作 | 以最先触发的有效操作为准 |
| 极低电量关机 | 关机前保存最后状态，Type-C 应急供电后自动恢复 |
| 指纹磨损/受伤 | 引导使用密码或人脸备用 |
| -25°C 低温 | 电池放电能力下降，电机扭矩保障 ≥ 2.0 N·m |
| 强光直射 | 人脸识别红外补光对抗强逆光，正常工作 |

## 6. 界面原型

### 6.1 App 主要页面

**首页**：门锁状态（在线/离线/电量/锁状态）→ 快捷操作（远程开锁/临时密码/记录）→ 最近动态

**开锁记录**：时间线 + 开锁方式图标 + 用户昵称 + 抓拍缩略图；筛选：时间/方式/用户

**设置**：用户管理 → 开锁方式管理 → 自动上锁延时 → 告警开关 → 场景联动 → 固件升级

### 6.2 门锁面板交互

**待机**：LED 指示呼吸灯(白=正常/红=低电量/橙=告警)  
**唤醒**：触摸面板 → 键盘背光亮 → 指纹灯亮 → 摄像头启动  
**成功**：绿灯 + "开锁成功"播报 → 电机声  
**失败**：红灯闪烁 + "验证失败" + 显示剩余次数

## 8. 技术约束

### 8.1 平台限制
- 门锁电池供电，平均功耗预算 ≤ 120μA（深度睡眠）
- SRAM 总量 ≤ 520KB（含协议栈），需严格控制任务栈大小
- WiFi 模块发射功率 ≤ 20dBm（SRRC 合规）

### 8.2 第三方依赖
- 人脸识别算法 SDK（虹软/商汤，需 ARM Cortex-M4+ 算力）
- 指纹识别模组驱动（FPC/汇顶）
- 推送服务：APNs + 国内推送通道（华为/小米 Push）
- 云服务：阿里云 IoT / AWS IoT（按量付费）

### 8.3 合规要求
- 通过 GA 374-2019 电子防盗锁检测
- 通过 CCC 强制认证（含生物识别模块）
- 通过 SRRC 无线电核准
- 符合《个人信息保护法》(PIPL) — 生物特征数据本地存储

## 12. 硬件需求规格

> 详细规范参见 [hardware-spec-template.md](references/hardware-spec-template.md)

### 12.1 主控方案

| 维度 | 需求分析 | 结论 |
|------|----------|------|
| 通信需求 | WiFi 2.4GHz + BLE 5.0 双模 | ESP32-S3 |
| 算力需求 | 人脸识别需 ARM Cortex-M4+ @ 240MHz + AI 加速 | ESP32-S3 (向量指令) |
| 内存需求 | 协议栈 + 应用 + 人脸模型 ≈ 400KB RAM | 内置 512KB SRAM + 8MB PSRAM |
| Flash | Boot + App A/B + NVS + 人脸模型 ≈ 10MB 需求 | 外挂 16MB SPI Flash |
| 功耗约束 | 深度睡眠 ≤ 80μA（电池供电） | ESP32-S3 deep sleep ~7μA, 模块整体 ~50μA |
| GPIO 需求 | 触摸 IC×1, 指纹模组×1, 摄像头×1, 电机×1, LED×3, 蜂鸣器×1, 门磁×1 | ≈ 25 GPIO |

### 12.2 外设选型

| 序号 | 器件 | 型号 | 接口 | 供电 | 关键参数 |
|------|------|------|------|------|----------|
| 1 | 主控模组 | ESP32-S3-WROOM-1 | — | 3.3V | 双核 240MHz, 512KB SRAM, 8MB PSRAM |
| 2 | 外挂 Flash | W25Q128JV | QSPI | 3.3V | 16MB |
| 3 | 指纹模组 | FPC1020A | SPI@8MHz | 3.3V | 508DPI, EER<0.1%, 活体检测 |
| 4 | 人脸摄像头 | GC02M2(IR) + GC032A(RGB) | DVP | 3.3V/1.8V | 双目 200W+30W, IR 补光 |
| 5 | 触摸 IC | SC12A | I2C(0x5A) | 3.3V | 12 通道电容式 |
| 6 | 电机驱动 | DRV8837 | PWM+GPIO | 5V | 1.8A 峰值, 内置 H 桥 |
| 7 | 门磁传感器 | AH9246(霍尔) | GPIO 中断 | 3.3V | 微功耗 1.8μA |
| 8 | 安全芯片 | ATECC608A | I2C | 3.3V | 密钥安全存储, ECC 签名 |

### 12.3 电源方案

**日耗电模型：**

| 工作模式 | 电流 | 时长/频率 | 日耗电(mAh) |
|----------|------|-----------|-------------|
| Deep Sleep | 50μA | 23.5h | 1.18 |
| BLE 广播 | 5mA | 0.1h/天 | 0.5 |
| 指纹采集+匹配 | 80mA | 10次×1.5s | 0.33 |
| 人脸采集+匹配 | 150mA | 5次×2s | 0.42 |
| 电机开锁 | 500mA | 10次×0.5s | 0.69 |
| WiFi 连接(上报/远程) | 120mA | 0.3h/天 | 36 |
| **合计** | — | — | **≈ 39.1** |

**电池选型：** 8 节 AA 碱性电池 × 2500mAh × 1.5V = 30,000mWh  
可用容量(80%) = 24,000mWh  
实际续航 = 24,000 / (39.1 × 3.7V) ≈ **165 天 ≈ 5.5 个月**

> 结论：碱性电池方案续航不足，需切换为 **5000mAh 锂电池 × 2 并联（10000mAh @ 3.7V = 37,000mWh）** 或优化 WiFi 连接频次。

**优化后模型（WiFi 按需连接，日均 0.1h）：**
日耗电 ≈ 39.1 - 24 = **15.1mAh**  
续航 = 24,000 / (15.1 × 3.7) ≈ **430 天**

**电源树：**
```
锂电池(2P1S, 3.7V 10000mAh) → 保护电路(BMS)
  ├── 3.3V Buck(LMR33630) → MCU + 指纹 + 触摸 + 摄像头 + Flash
  ├── 5V Boost(TPS61023)   → 电机驱动(仅开锁时使能, 平时 EN=0)
  └── Type-C 5V 输入       → 充电 IC(IP2312, 1.5A)
```

### 12.4 射频设计

- **天线**：板载 PIFA PCB 天线，净空区 ≥ 5mm × 15mm，π 型匹配预留
- **传导测试**：预留 U.FL 测试座
- **认证**：SRRC 已确认（ESP32-S3 模组已有 SRRC，整机无需重复）

### 13.1 物模型（核心属性）

| 属性ID | 名称 | 类型 | 读写 | 说明 |
|--------|------|------|------|------|

### 13.2 MQTT Topic 清单

| Topic | QoS | 说明 |
|-------|-----|------|
| `smartlock/{did}/up/property/report` | 1 | 属性上报 |
| `smartlock/{did}/down/property/set` | 1 | 属性设置 |
| `smartlock/{did}/up/event/post` | 1 | 事件上报 |
| `smartlock/{did}/down/service/invoke` | 1 | 服务调用 |
| `smartlock/{did}/up/service/response` | 1 | 服务响应 |
| `smartlock/{did}/down/ota/notify` | 1 | OTA 通知 |
| `smartlock/{did}/up/ota/progress` | 1 | OTA 进度 |

### 13.3 安全设计

- 设备一机一密（device_secret 预烧录在 ATECC608A 安全芯片中）
- MQTT CONNECT: username=device_id, password=HMAC(device_secret, timestamp)
- 远程开锁：token + timestamp 签名，TTL=30s，防重放
- OTA 固件 HTTPS 下载 + SHA256 校验 + ECDSA 签名验签

## 14. 固件架构设计

> 详细规范参见 [firmware-architecture-template.md](references/firmware-architecture-template.md)

### 14.1 运行环境

- **RTOS**：FreeRTOS（ESP-IDF 原生集成）
- **SDK**：ESP-IDF v5.2
- **编译工具链**：xtensa-esp32s3-elf-gcc

### 14.2 Flash 分区（16MB）

```
0x000000  Bootloader         (64KB, 写保护)
0x010000  OTA Data           (8KB)
0x012000  NVS                (512KB, 配网+用户数据)
0x092000  App Firmware (A)   (5MB, 运行区)
0x592000  App Firmware (B)   (5MB, OTA 区)
0xA92000  Face Model Data    (3MB, 人脸特征)
0xD92000  Factory Data       (64KB, 设备证书+SN+校准)
0xDA2000  Scratch/Temp       (2.4MB, 剩余)
```

### 14.3 任务划分

| 任务 | 优先级 | 栈 | 职责 |
|------|--------|-----|------|
| sys_monitor | 5(最高) | 2KB | 看门狗、电量检测、系统健康 |
| comm_mqtt | 4 | 6KB | MQTT 连接/心跳/消息收发 |
| face_recog | 3 | 16KB | 人脸采集→检测→识别(耗时最长) |
| finger_sensor | 3 | 4KB | 指纹图像采集+匹配 |
| motor_ctrl | 3 | 2KB | 电机 H 桥控制 |
| touch_key | 2 | 2KB | 触摸面板扫描 |
| led_audio | 2 | 2KB | LED 指示+语音播报 |
| ota_task | 2 | 8KB | OTA 下载+校验+升级 |
| idle_log | 1 | 2KB | 日志批量上报 |

### 14.4 低功耗状态机

```
DEEP_SLEEP(50μA) → 触摸/门磁中断唤醒 → ACTIVE(~80mA)
  → 开锁完成/通信完成 → 空闲 60s → LIGHT_SLEEP(0.8mA)
  → 空闲 5min → DEEP_SLEEP
```

**策略：**
- 触摸/门磁 GPIO 中断唤醒（功耗 <1μA）
- WiFi 仅在有数据上报或远程指令时连接，其余时间关闭
- BLE 广播占空比：前 30s 间隔 100ms → 后降为 1s
- 摄像头仅在唤醒和识别时上电

### 14.5 OTA 升级

- **双分区 A/B**：下载到非运行分区，校验后切换
- **安全**：固件包 ECDSA 签名 + HTTPS 下载 + SHA256 校验
- **回滚**：新固件启动失败 3 次 → bootloader 自动切回旧分区
- **防回滚**：版本号单调递增，bootloader 拒绝低版本启动
- **策略**：仅 WiFi 连接时下载，电量 >30%，凌晨 2-5 点优先
